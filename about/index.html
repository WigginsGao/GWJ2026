紧急情况

作为城市的紧急救援团队负责人，你将获得一张你所在国家的特殊地图。

该地图显示了一些通过道路连接的分散城市，道路是双向的。

地图上标出了每个城市的救援队数量以及每对城市之间的每条道路的长度。

当其他城市发出紧急求援信息时，你的工作是尽快带领你的士兵前往该地点，同时，在途中尽可能多地调动救援帮手。

输入格式

第一行包含四个整数 N，表示城市数量（城市编号从 0 到 N−1），M 表示道路数量，C1 表示你当前所在的城市编号，C2 表示发出紧急求援信息的城市编号。

第二行包含 N 个整数，其中第 i 个整数表示城市 i 的救援队数量。

接下来 M 行，每行包含三个整数 c1,c2,L**i，表示城市 c1 和城市 c2 之间存在一条道路相连，道路长度为 L**i。

数据保证 C1 和 C2 之间至少存在一条路径相连。

输出格式

共一行，两个整数，第一个整数表示 C1 和 C2 之间最短路的数量，第二个整数表示走最短路的情况下，能聚集到的救援队最大数量。

数据范围

2≤N≤500,
1≤M≤600,
1≤L**i≤200,
每个城市包含的救援人员数量不超过 200。

输出格式

判断二叉搜索树

二叉搜索树 (BST) 递归定义为具有以下属性的二叉树：

  • 若它的左子树不空，则左子树上所有结点的值均小于它的根结点的值
  • 若它的右子树不空，则右子树上所有结点的值均大于或等于它的根结点的值
  • 它的左、右子树也分别为二叉搜索树

按位相除

在进行除法运算时，从高位到低位依次处理每位数字，将上次余数 ×10+当前位作为被除数，继续进行除法运算。按位相除，每次的商值范围均为 0~9。

例如，求解 129786/15，过程如下：

  1. 将高精度数正序存储在数组 a[] 中，单精度数 b=15。

  2. 从高位到低位依次处理每位数字，将结果存储在数组 c[] 中。

  • i=1：初始时，余数 x=0，x=x×10+a[1]=1，c[1]=x/15=0，更新余数 x，x=x%15=1。
  • i=2：x=x×10+a[2]=12，c[2]=x/15=0，更新余数 x，x=x%15=12。
  • i=3：x=x×10+a[3]=129，c[3]=x/15=8，更新余数 x，x=x%15=9。
  • i=4：x=x×10+a[4]=97，c[4]=x/15=6，更新余数 x，x=x%15=7。
  • i=5：x=x×10+a[5]=78，c[5]=x/15=5，更新余数 x，x=x%15=3。
  • i=6：x=x×10+a[6]=36，c[6]=x/15=2，更新余数 x，x=x%15=6。
  1. 删除前导 0，从高位到低位依次输出答案 8652，即 129786/15=8652。

Qt 框架下解决按钮点击误触发窗口拖拽的问题

发表于 2026-02-26 更新于 2026-06-01 分类于 Qt

问题现象

在使用 Qt 开发桌面应用时，发现点击 BtForm 上的按钮时，偶尔会导致主窗口意外移动。

根因分析

经过排查，发现问题源于 Qt 的事件处理机制：

  1. 事件序列问题：当用户点击按钮时，如果手指有轻微抖动或滑动，Qt 框架可能将一次”点击”事件拆分为：

    • 按钮按下（mousePressEvent）
    • 鼠标移动（mouseMoveEvent）
    • 按钮释放（mouseReleaseEvent）

  2. 处理速度因素：CPU 处理速度极快，人眼只能感知到一次单击，但底层事件可能已被解析为”点击 + 拖动”的组合，从而触发了窗口移动逻辑。

解决方案

核心思路：在点击 BtForm 按钮时，临时禁用窗口拖拽功能。

场景 操作 isDrag 状态
鼠标在非 BtForm 区域按下 准备拖拽 true
鼠标在 BtForm 按钮上按下 禁止拖拽 false
鼠标移动时 仅当 isDrag == true 才移动窗口 —

总结

通过在 BtForm 和 QLingMusic 之间建立信号-槽通信，精确识别用户点击意图，有效区分”点击按钮”和”拖拽窗口”两种操作，解决了误触发问题。

关键字

static

C语言中的用法：

  1. 修饰局部变量：一般的局部变量是存储在栈区的，出了作用域之后就会销毁，但是经过static修饰后的局部变量存储在静态区（全局区），直到程序结束后变量才会销毁，生命周期会变长。（不影响作用域）
  2. 修饰全局变量： 一般的全局变量是具有外部链接属性的，在同一工程下的任何.c文件中都是可见，static 修饰全局变量，会将它的链接属性由外部链接变成内部链接，只能在当前的 .c文件中可见。
  3. 修饰函数： 函数本身也是具有外部链接属性的， 但是被static修饰后只能在当前源文件中使用

C++中用法：

  1. 上述C语言中国用法同样适用。
  2. 修饰成员变量： 被 static 修饰的成员变量叫做静态成员变量，属于当前类的资源，被该类创建出的所有对象同时共享，被static 修饰的成员变量在类声明的时候不能给缺省值（这是需要注意的点），需要再类外面给出定义。因为缺省值是给初始化列表用的，静态成员变量不走初始化列表 。
  3. 修饰成员函数 ：被 static 修饰的成员函数叫做静态成员函数，该函数中没有隐藏的 this 指针，所以在函数体中无法访问类中的任何非静态成员（成员变量和成员函数） 。

const关键字的作用

被它修饰的值不能改变，是只读变量。必须在定义的时候就给它赋初值。

关于多态

多态分为哪几类？

类型 名称 实现方式 绑定时机
静态多态 编译时多态 函数重载、运算符重载、模板 编译期
动态多态 运行时多态 虚函数（virtual） 运行期

多态实现的原理

C++中的多态性是通过虚函数（virtual function）和虚函数表（vtable）来实现的。 运行时通过 虚表指针 查表，决定调用哪个函数。

说说C++内存分区

  1. 栈区
  • 存储：局部变量、函数参数、返回地址
  • 生命周期：随函数调用创建，随函数返回销毁
  • 特点：自动分配释放，速度快，空间较小
  1. 堆区
  • 存储：new/malloc 动态分配的内存
  • 生命周期：手动 delete/free 释放
  • 特点：灵活可控，空间大，需手动管理（易内存泄漏）
  1. 全局/静态区
  • 存储：全局变量、static 静态变量
  • 生命周期：程序启动分配，程序结束释放
  • 细分：已初始化数据区、未初始化数据区（BSS）
  1. 常量区
  • 存储：字符串常量、const 修饰的常量
  • 生命周期：程序开始到结束
  • 特点：只读，不可修改
  1. 代码区
  • 存储：可执行代码（机器指令）
  • 生命周期：程序开始到结束
  • 特点：只读，可被多个进程共享

关于C++的智能指针

智能指针的思想是什么？

用对象管理资源（RAII），自动释放内存，防止内存泄漏。

智能指针类型有啥？

  • unique_ptr（独占）
  • shared_ptr（共享）
  • weak_ptr（弱引用，解决循环引用）

啥是作用域？

标识符（变量、函数、类等）在代码中可见、可访问的范围。

出函数时候不是出作用域了吗？为什么还能拿到？

作用域和函数周期不一样

static 局部变量是两回事分开：

  1. 作用域：只在函数内可见（编译器限制访问）
  2. 生命周期：存在静态区，程序结束才销毁

作用域管”能不能看见”，生命周期管”还在不在”。static 让变量”看不见但还在”。
